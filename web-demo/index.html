Simulador Web do Fluxo do Projeto (Main → Controller → Service → Repository → Entity)

Visual educacional mostrando quando o sistema “entra no repositório” e quando a “Entity entra na linha de raciocínio” (por exemplo, quando o Controller monta o Cliente e envia para o Service).

Execução simulada
Main
Orquestra e chama o menu do Controller.
Controller
Monta a Entity (ex: new Cliente(...)) e chama clienteService.salvar(entity).
Service
Valida regras e pode interromper com throw antes do Repository.
Repository
Entrada no repositório: RepositoryGenerico.salvar() grava no Map e (se ID for null) gera o ID.
Entity
Após o Repository, a Entity passa a ter ID preenchido.
Log
Aguardando...
Legenda
Map/IDs e validações seguem a ideia do projeto Java:
  • Controller → cria Entity
  • Service → valida e pode throw
  • RepositoryGenerico → se id==null, atribui proximoId e faz dados.put
